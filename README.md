# 🍌 AI - Image Generation Service

Полноценный асинхронный микросервис для генерации изображений с помощью нейросетей (Google Gemini Image 3.1 Flash), построенный на микросервисной архитектуре с использованием брокера сообщений.

Проект разработан в рамках практического блока курса по ML Engineering и демонстрирует production-ready подход к созданию backend-приложений.

## 🛠 Технологический стек

*   **Backend API**: FastAPI, Pydantic, Python 3.10+
*   **База данных**: PostgreSQL, SQLAlchemy (ORM)
*   **Асинхронные очереди**: RabbitMQ, Pika
*   **Frontend**: HTML5, Bootstrap 5, Vanilla JS, Server-Side Rendering (Jinja2)
*   **Инфраструктура**: Docker, Docker Compose
*   **Тестирование**: Pytest, FastAPI TestClient
*   **ML Интеграция**: Google GenAI (модель `gemini-3.1-flash-image-preview`)

## 🚀 Архитектура системы

1. **FastAPI (Publisher)**: Принимает запросы от пользователей (через Web UI или Swagger), списывает кредиты с баланса, создает запись в БД и публикует задачу в RabbitMQ.
2. **RabbitMQ (Broker)**: Гарантирует сохранность сообщений (durable queues) и балансирует нагрузку между воркерами по принципу Round-robin (`prefetch_count=1`).
3. **Workers (Consumers)**: Изолированно работают в Docker-контейнерах. Получают задачи, делают запросы к Google Gemini API, конвертируют бинарный ответ в `.png` через `Pillow`, сохраняют локально и обновляют статусы в БД.
4. **Безопасность и Авторизация**: Реализована JWT-авторизация (Bearer). Для Web-интерфейса токен надежно сохраняется в Cookies. Пароли пользователей хешируются алгоритмом `bcrypt`.

## ⚙️ Инструкция по локальному запуску

Для запуска проекта на вашем компьютере должны быть установлены **Python 3.10+** и **Docker**.

### Шаг 1. Настройка окружения
Создайте файл `.env` в корневой папке проекта на основе шаблона `.env.example`:
```bash
cp .env.example .env
```
⚠️ **Важно:** Откройте созданный файл `.env` и обязательно укажите ваш ключ от Google AI Studio в переменной `GEMINI_API_KEY` (при наличии). Остальные настройки БД можно оставить по умолчанию.

### Шаг 2. Запуск инфраструктуры (БД, RabbitMQ и Воркеры)
Поднимите контейнеры в фоновом режиме. Мы запускаем сразу 2 воркера для демонстрации балансировки нагрузки:
```bash
docker compose up -d --build --scale worker=2
```

### Шаг 3. Установка зависимостей и Инициализация БД
Создайте виртуальное окружение и установите библиотеки:
```bash
python -m venv venv

# Активация для Windows:
venv\Scripts\activate
# Активация для Mac/Linux:
source venv/bin/activate

# Установка зависимостей
pip install -r app/requirements.txt
```

Запустите скрипт инициализации. Он автоматически создаст все таблицы в PostgreSQL и добавит демо-админа со стартовым балансом:
```bash
python app/src/init_db.py
```

### Шаг 4. Запуск Web-сервера (FastAPI)
Перейдите в папку с исходным кодом и запустите сервер Uvicorn:
```bash
cd app/src
uvicorn main:app --reload
```

## 🧪 Тестирование (E2E)

Проект покрыт автоматизированными сквозными тестами (End-to-End). Проверяется регистрация, авторизация, логика списания баланса (в т.ч. ошибки 402 при нехватке средств), защита от дубликатов и целостность истории аудита.

Для запуска тестов выполните (база данных в Docker должна быть запущена):
```bash
cd app/src
pytest tests/test_api.py -v
```
